# Slack Etiquette Skill

## When to Use

When composing Slack messages — DMs, channel posts, or thread replies.

## Core Principles

1. **Thread everything**: Top-level = announcements; discussion = threads
2. **Mention sparingly**: Only @-mention people who need to take action
3. **Scannable**: Bold key info, use bullets, keep it short
4. **Respect async**: Not everyone is online — write self-contained messages

## Patterns

### Announcement
```
📢 **[Topic]**: [One-line summary]

Details:
- [Point 1]
- [Point 2]

Action needed: [what / who / by when]
Thread for questions 👇
```

### Request
```
@name — Could you [specific ask]?

Context: [brief background]
Deadline: [when]
```

### Status Update
```
**[Project] Update** — [Date]
✅ Completed: [what]
🔄 In progress: [what]
🔴 Blocked: [what — and what help is needed]
```

## Anti-Patterns

- **@here / @channel abuse**: Reserve for truly urgent, time-sensitive items
- **Wall of text**: Top-level messages over 5 lines should be a thread or document
- **DM for group decisions**: If 3+ people need to know, use a channel
- **Reply in wrong thread**: Check you're replying in the right context
