# Hiring Evaluation Skill

## When to Use

When designing interviews, creating evaluation criteria, or assessing candidates.

## Scorecard Template

```markdown
## Candidate Scorecard: [Role]

### Must-Have Criteria
| Criterion | Weight | Score (1-5) | Notes |
|-----------|--------|-------------|-------|
| [Technical skill] | 5 | | |
| [Problem solving] | 4 | | |
| [Communication] | 3 | | |

### Nice-to-Have
| Criterion | Weight | Score (1-5) | Notes |
|-----------|--------|-------------|-------|

### Overall Assessment
- **Hire / No Hire / Strong Hire**
- **Key strengths**: [specific examples from interview]
- **Concerns**: [specific, with evidence]
- **Comparison to bar**: [Above / At / Below the hiring bar]
```

## Interview Design

### Structured Interview Principles
1. Same questions for all candidates (comparability)
2. Behavioral questions > hypothetical ("Tell me about a time..." > "What would you do if...")
3. Rubric defined before interviews begin
4. Multiple interviewers with different focus areas

### STAR Framework for Evaluation
- **Situation**: Context of the example
- **Task**: What they were responsible for
- **Action**: What they specifically did (look for "I" not "we")
- **Result**: Measurable outcome

## Anti-Patterns

- **Gut feeling hiring**: Use scorecards to reduce bias
- **Culture fit = like me**: Look for culture add, not culture clone
- **Halo effect**: One strong answer shouldn't compensate for multiple weak ones
- **Lowering the bar**: "We need someone now" is never a reason to compromise on quality
