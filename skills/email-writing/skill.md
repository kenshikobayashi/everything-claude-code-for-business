# Email Writing Skill

## When to Use

When composing any business email — new messages, replies, or forwards.

## Core Principles

1. **Purpose first**: State why you're writing in the first sentence
2. **One email, one ask**: Don't bundle unrelated requests
3. **Scannable**: Use bullets for 3+ items, bold key info
4. **Clear CTA**: End with exactly what you need the reader to do
5. **Appropriate tone**: Match formality to relationship and context

## Patterns

### Subject Line
- **Action request**: "[Action] [Topic] by [deadline]" → "Review Q1 budget by Friday"
- **FYI**: "[FYI] [Topic]" → "[FYI] Updated project timeline"
- **Decision**: "[Decision needed] [Topic]" → "[Decision needed] Vendor selection"
- **Reply**: Keep the original subject unless the topic has changed

### English Email Structure

```
Subject: [Action] + [Topic] + [Deadline if any]

Hi [Name],

[Purpose — 1 sentence: why you're writing.]

[Body — key information:]
- Point 1
- Point 2
- Point 3

[Ask — what you need from them, by when.]

[Closing — Thanks / Best regards / etc.]
[Name]
```

### Japanese Email Structure (社外フォーマル)

```
件名: [簡潔なトピック]

[会社名]
[部署名]
[名前]様

お世話になっております。[自社名]の[自分の名前]です。

[季節の挨拶 — 必要な場合]

[目的 — この連絡の理由を1文で]

[本文 — 箇条書きで要点を]
・[ポイント1]
・[ポイント2]

[依頼事項 — 何を・いつまでに]

お忙しいところ恐れ入りますが、ご確認のほどよろしくお願いいたします。

[署名]
```

### Japanese Email Structure (社内)

```
件名: [トピック]

[名前]さん

お疲れ様です。[自分の名前]です。

[目的 — 1文]

[本文]

よろしくお願いします。
[名前]
```

## Tone Calibration

| Context | English | Japanese |
|---------|---------|----------|
| Declining politely | "Unfortunately, we won't be able to..." | "誠に申し訳ございませんが、今回は見送らせていただきたく..." |
| Urgent request | "I need this by [time] because..." | "恐れ入りますが、[時間]までにご対応いただけますと幸いです。" |
| Following up | "Just checking in on..." | "先日の件について、その後いかがでしょうか。" |
| Bad news | "I want to let you know that..." | "ご報告がございます。" |

## Anti-Patterns

- **Wall of text**: No one reads 10-paragraph emails. Break it up or schedule a meeting.
- **Passive aggressive CC**: Don't CC someone's manager as a threat. Address issues directly.
- **Reply-all default**: Only reply-all if everyone needs to see your response.
- **Burying the ask**: Put the request in the first 3 lines, not the last paragraph.

## Checklist

- [ ] Subject line is specific and includes action if needed
- [ ] Purpose is stated in the first sentence
- [ ] Ask is clear (what, who, by when)
- [ ] Tone matches the relationship
- [ ] No confidential information that shouldn't be in this email
- [ ] Correct recipients (To/CC/BCC)
