# Data Storytelling Skill

## When to Use

When presenting data to an audience — choosing visualizations, structuring narratives, extracting insights.

## The Data Story Framework

### What → So What → Now What
1. **What**: State the fact/finding (the data)
2. **So What**: Why it matters (the insight)
3. **Now What**: What to do about it (the action)

Example:
- **What**: "Churn increased from 3% to 5% in Q4"
- **So What**: "At this rate, we lose $500K ARR annually and our LTV:CAC falls below 2:1"
- **Now What**: "Prioritize retention initiative: customer success outreach for at-risk accounts"

## Visualization Selection

| Data Type | Best Chart | When to Use |
|-----------|-----------|-------------|
| Comparison (few items) | Bar chart | Comparing categories |
| Comparison (many items) | Table with conditional formatting | Detailed comparison |
| Trend over time | Line chart | Showing change/direction |
| Part of whole | Pie/donut (max 5 slices) | Showing proportions |
| Distribution | Histogram | Showing spread/frequency |
| Relationship | Scatter plot | Showing correlation |
| Ranking | Horizontal bar | Sorted comparison |

## Narrative Principles

1. **Lead with the insight**, not the methodology
2. **One chart, one message**: Don't overload visualizations
3. **Annotate key points**: Circle, highlight, or label the important thing
4. **Compare against something**: Absolute numbers need context (target, last period, benchmark)
5. **Acknowledge limitations**: "This data covers X period and Y segment"

## Anti-Patterns

- **Chart junk**: 3D effects, excessive gridlines, unnecessary decoration
- **Misleading axes**: Starting Y-axis at non-zero to exaggerate changes
- **Cherry-picking**: Showing only the data that supports your conclusion
- **Correlation = causation**: "A and B both increased" doesn't mean A caused B
