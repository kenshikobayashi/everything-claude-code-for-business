# Output Quality Rules

## Always

- Include an executive summary for any document longer than 1 page
- Action items must have three elements: owner, due date, and specific description
- Data claims must reference their source
- Proposals must address risks and mitigations
- Reports must include comparison context (target, previous period, or benchmark)
- Use consistent formatting throughout a document

## Structure

- Use headers to break up content — no wall of text longer than 5 paragraphs
- Tables for structured comparisons; bullets for lists; paragraphs for narrative
- Bold key terms, decisions, and names on first mention

## Numbers

- Always include units (¥, $, %, hours, people)
- Round appropriately for audience: executives (millions), operators (thousands)
- Totals must add up — verify arithmetic
- Show period-over-period change (absolute and percentage)

## Language Quality

- No orphaned acronyms — define on first use
- Avoid jargon unless the audience is technical
- Spell-check proper nouns (company names, product names, people's names)
- Consistent terminology — don't alternate between synonyms for the same concept

## Never

- Present unverified claims as facts
- Use vague qualifiers ("significant", "substantial") without quantification
- Leave placeholders ([TBD], [TODO]) in final outputs
- Generate content that contradicts data provided by the user
