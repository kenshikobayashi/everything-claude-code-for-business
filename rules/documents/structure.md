# Document Structure Rules

## All Documents

- Start with an executive summary (max 100 words for proposals, 3-5 sentences for reports)
- Use descriptive headers (not "Section 1" but "Market Analysis Results")
- One idea per paragraph
- Use tables for structured data, bullets for lists, paragraphs for narrative

## Proposals

- Must include: Executive Summary, Problem, Solution, Alternatives, Outcomes, Resources, Risks, Ask
- Always compare at least 3 options including "do nothing"
- Quantify all outcomes

## Reports

- Must include: Summary, Data/Findings, Analysis, Recommendations, Next Steps
- All metrics need comparison context (target, benchmark, previous period)
- Include trend indicators (↑↓→)

## Meeting Minutes

- Must include: Decisions, Action Items, Discussion Summary, Next Meeting
- Action items must have: owner, due date, specific description
- No action item should say "the team" — name a single person

## SOPs

- Must include: Purpose, Roles, Prerequisites, Procedure, Exceptions, Checklist
- Use numbered steps for sequential procedures
- Mark all decision points explicitly
