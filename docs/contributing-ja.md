# コントリビューションガイド

Everything Claude Code for Business に新しいエージェント・スキル・ルールを追加する方法です。

## ファイルフォーマット

### エージェント

エージェントは YAML frontmatter 付きのMarkdownファイルとして定義します。

**配置場所**: `agents/`

**フォーマット**:

```markdown
---
name: agent-name
description: このエージェントの目的を1行で
tools: ["Read", "Grep", "WebFetch", "WebSearch"]
model: sonnet
---

# エージェント名

## 役割
エージェントの目的と専門分野の明確な説明。

## 機能
- できること
- 使用するツール
- 生成する出力

## ワークフロー
1. ステップ 1: ...
2. ステップ 2: ...
3. ステップ 3: ...

## 出力フォーマット
期待される出力構造の説明。

## 制約
- してはいけないこと
- 境界と制限
```

**命名規則**: `lowercase-with-hyphens.md`（例: `meeting-facilitator.md`）

---

### スキル

スキルは、すべての機能を統一するコンポーネントです。スラッシュコマンド（例: `/briefing`）とドメイン知識パッケージ（例: email-writing）の両方がスキルとして `skills/{skill-name}/SKILL.md` に配置されます。

**配置場所**: `skills/{skill-name}/SKILL.md`

**フォーマット**:

```markdown
---
name: skill-name
description: 1行の説明
triggers:
  - このスキルを有効化するキーワードやパターン
---

# スキル名

## いつ使うか
このスキルが適用される状況の説明。

## 基本原則
- 原則 1
- 原則 2

## パターン

### パターン名
**いつ**: 状況の説明
**どのように**: 実装アプローチ
**例**:
```
具体例
```

## アンチパターン
- 避けるべきこととその理由

## チェックリスト
- [ ] 品質チェック項目 1
- [ ] 品質チェック項目 2
```

**命名規則**: ディレクトリ名 = スキル名、内部は常に `SKILL.md`（例: `skills/email-writing/SKILL.md`）

---

### ルール

ルールは常にアクティブな動作制約ガイドラインです。

**配置場所**: `rules/{category}/`

**フォーマット**:

```markdown
# ルールカテゴリ: ルール名

## 常に
- これをする
- あれをする

## 絶対にしない
- これをしない
- あれをしない

## [状況]の場合
- この特定の動作を適用
```

**命名規則**: `lowercase-with-hyphens.md`（例: `rules/common/communication-style.md`）

---

## 新しいコンポーネントの追加

### 1. 新しいスキル（スラッシュコマンドまたはドメイン知識）

```bash
# スキルディレクトリとファイルを作成
mkdir -p skills/my-skill
touch skills/my-skill/SKILL.md
```

frontmatter を含めて編集:
```markdown
---
name: my-skill
description: このスキルの1行説明
---

# スキルの指示...
```

テスト（スラッシュコマンドの場合）:
```
claude
> /my-skill "テスト引数"
```

### 2. 新しいエージェント

```bash
touch agents/my-agent.md
```

### 3. 新しいルール

```bash
touch rules/common/my-rule.md
```

## 品質チェックリスト

PR を提出する前に:

- [ ] **命名**: ファイルは `lowercase-with-hyphens` を使用
- [ ] **Frontmatter**: 必須フィールド（`name`、`description`）を含む有効な YAML
- [ ] **言語**: 該当する場合は英語 + 日本語版を作成
- [ ] **機密情報なし**: API キー、トークン、個人情報を含まない
- [ ] **下書きポリシー**: スキルは自動送信しない — 常に下書きを作成
- [ ] **テスト済み**: CLI と Desktop の両方でスキルが動作
- [ ] **ドキュメント**: 新しいスキルを追加した場合、リファレンスドキュメントを更新

## コードスタイル

- Markdown ファイル: 末尾の空白なし、EOF に改行1つ
- YAML frontmatter: 2スペースインデント
- エージェント/スキルの説明: 命令形（「レポートを生成する」→「レポートを生成」）
- 日本語テキスト: 標準的なビジネス日本語を使用

```markdown
## テスト
- [ ] CLI でテスト済み

## チェックリスト

- [ ] 有効な frontmatter がある
- [ ] ハードコードされた個人情報がない

- [ ] リファレンスドキュメントを更新済み
```
