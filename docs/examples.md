# Examples — Role-Based CLAUDE.md Templates

Ready-to-use CLAUDE.md templates for common business roles. Copy the one closest to your role and customize.

---

## Startup CEO / Founder

```markdown
# Business Context

## User
- Name: Jane Smith
- Email: jane@startup.com
- Company: StartupCo (Series A, 25 people)
- Role: CEO / Co-founder
- Timezone: America/San_Francisco

## MCP Integrations
- Gmail: jane@startup.com
- Google Calendar: primary
- Slack: startupco-team
- Notion: Board DB, OKR DB, Hiring Pipeline

## Preferences
- Language: English
- Email tone: Professional but warm
- Draft policy: Always create drafts, never auto-send
- Meeting minutes: Always capture investor commitments separately
- Priority signals: Investor emails → customer emails → team → vendors

## Key Context
- Current fundraise: Series B (targeting Q2)
- Board meetings: Monthly, last Thursday
- All-hands: Bi-weekly Friday 16:00
- Investor update: Monthly, first Monday

## Communication Rules
- Investor communications: Formal, data-driven, always include metrics
- Team communications: Transparent, motivating, concise
- Customer escalations: Empathetic, solution-oriented
- Board updates: Structured as Wins / Challenges / Asks

## Trigger Words
- "board" → Prepare board update format
- "investor" → Use investor communication tone
- "fundraise" → Reference current round context
```

## User
- Name: Sarah Kim

- Role: Senior Product Manager, Growth Squad

## MCP Integrations
- Gmail: sarah.kim@product.io

- PRD format: Problem → Hypothesis → Solution → Metrics → Timeline
- Decision log: Always document decisions with rationale
- Priority framework: RICE (Reach × Impact × Confidence / Effort)

## Stakeholders
- Engineering: Alex (EM), Team of 6
- Design: Mike (Lead Designer)
- Data: Carol (Data Analyst)
- Sales: Tom (Sales Lead, feature requests)

- PRD reviews: Share draft → 48h async feedback → sync discussion
- Feature requests from Sales: Log in Notion, prioritize in weekly grooming
- Customer escalations: Acknowledge within 4h, resolution plan within 24h
- Stakeholder updates: Weekly Friday email, monthly all-hands presentation

## Key Metrics
- North star: Weekly Active Users (WAU)
- Secondary: Activation rate, Retention D7/D30, NPS
- Growth squad OKR: +15% WAU by Q2

## Consultant

```markdown
# Business Context

## User
- Name: David Park
- Email: david.park@consulting.com
- Company: ConsultingCo (Strategy & Operations)
- Role: Senior Consultant
- Timezone: Asia/Tokyo

## MCP Integrations
- Gmail: david.park@consulting.com
- Google Calendar: primary
- Slack: consultingco
- Notion: Client DB, Deliverable Tracker, Template Library

## Preferences
- Language: English / 日本語 (bilingual deliverables common)
- Email tone: Formal, authoritative
- Draft policy: Always create drafts, never auto-send
- Deliverable format: Pyramid principle (conclusion first)
- Slide design: McKinsey-style (action title, evidence body)

## Current Engagements
- Client A: Market entry strategy (Japan) — 8 weeks remaining
- Client B: Operational efficiency review — 3 weeks remaining
- Business development: 2 proposals in pipeline

## Communication Rules
- Client communication: Always via engagement manager
- Internal: Direct Slack for quick questions, email for decisions
- Proposal format: Situation → Complication → Resolution
- Status reports: Weekly to engagement manager, bi-weekly to client

## Quality Standards
- All deliverables: Peer review before client submission
- Data: Triple-check all numbers, cite all sources
- Visuals: Consistent brand colors, no clip art
- Language: No jargon without definition, audience-appropriate complexity
```

- Slack User ID: U0EXAMPLE01

- Google Calendar: primary (Asia/Tokyo)
- Slack: techco-workspace ワークスペース (U0EXAMPLE01)

## Preferences
- Language: 日本語
- メールトーン: ビジネス敬語（デフォルト）
- Slackトーン: 丁寧語（デフォルト）
- 下書きポリシー: 常に下書き作成、自動送信禁止
- 議事録フォーマット: 決定事項 → AI → 議論
- 報告書スタイル: エグゼクティブサマリー先頭

## 敬語ルール
- 社内（上司）: 丁寧語 + 尊敬語
- 社内（同僚）: 丁寧語
- 社外（クライアント）: 尊敬語 + 謙譲語
- 社外（ベンダー）: 丁寧語 + 尊敬語

## チームコンテキスト
- 定例: 毎日 09:00 スタンドアップ（15分）
- スプリント: 2週間、月曜開始
- 週次レビュー: 金曜 16:00

## トリガーワード
- 「おはよう」「今日」「ブリーフィング」→ /briefing 実行
- 「振り返り」→ /review 実行
- 「週次レビュー」→ /weekly 実行

## コミュニケーションルール
- メール・Slackは必ず「下書き」作成。自動送信禁止
- クライアント連絡: 必ずPMをCC
- 緊急連絡: Slack DM → 電話
- 非緊急: メールまたはSlackスレッド
