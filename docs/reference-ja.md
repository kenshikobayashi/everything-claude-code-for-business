# スキルリファレンス

Everything Claude Code for Business の全エージェント・スキル（スラッシュコマンド含む）の詳細リファレンスです。

---

## エージェント

エージェントは、Claude がタスクを委任する特化型サブエージェントです。自律的に実行し、結果を返します。

### chief-of-staff（チーフ・オブ・スタッフ）

**目的**: 中央トリアージエージェント — 情報の優先度判断、タスクのルーティング、日常ワークフロー管理を行います。

**機能**:
- 受信メッセージ（Gmail + Slack）を4段階に分類：スキップ / 情報のみ / 会議情報 / 要対応
- カレンダー・デッドライン・依存関係を考慮したタスク優先度の再計算
- プロジェクト切替時の関連コンテキスト自動読み込み
- 集中時間と会議時間のバランス提案

**呼び出し元**: `/briefing`、`/inbox-triage`、`/task-prioritize`

---

### meeting-facilitator（会議ファシリテーター）

**目的**: 会議の設計・準備・フォローアップを担当します。

**機能**:
- 会議準備パッケージの生成（アジェンダ、論点、背景情報）
- 会議メモの構造化（決定事項、アクションアイテム、次のステップ）
- フォローアップタスクの抽出と通知メッセージの下書き

**呼び出し元**: `/meeting-prep`、`/meeting-minutes`、`/meeting-followup`

---

### business-analyst（ビジネスアナリスト）

**目的**: データを分析し、構造化されたビジネスインサイトを生成します。

**機能**:
- 分析フレームワークの適用（SWOT、ポーターの5力、BCGマトリクス）
- 意思決定支援のための加重スコアリングマトリクス作成
- 生データからアクショナブルなナラティブへの変換

**呼び出し元**: `/data-insight`、`/decision-matrix`、`/market-scan`

---

### strategy-advisor（戦略アドバイザー）

**目的**: 戦略立案と長期的思考の支援。

**機能**:
- シナリオプランニングとリスク評価
- OKR策定とアラインメント
- 戦略オプションの評価

**呼び出し元**: `/okr-update`、`/risk-check`

---

### communication-drafter（コミュニケーション起草者）

**目的**: 適切なトーンとコンテキストでビジネスコミュニケーションを下書きします。

**機能**:
- 敬語レベルを調整可能なメール下書き
- スレッドコンテキストを意識したSlackメッセージ作成
- 受信者との関係性に基づくトーン調整
- 日本語の敬語（尊敬語・謙譲語・丁寧語）対応

**呼び出し元**: `/draft-email`、`/draft-slack`、`/feedback-draft`

---

### document-reviewer（ドキュメントレビューアー）

**目的**: 生成されたすべてのビジネス文書の品質ゲート。

**機能**:
- 論理的一貫性チェック（主張と根拠の整合性）
- トーンの適切性（対象読者に応じた敬語レベル）
- 機密情報の検出
- アクショナビリティの確認（読者にとって次の行動が明確か）

**呼び出し元**: 品質ゲート有効時、`/draft-*` コマンドの後に自動実行

---

### project-tracker（プロジェクト追跡者）

**目的**: プロジェクトの健全性を監視し、ステータスレポートを生成します。

**機能**:
- Notion / Linear からタスク状況を集約
- ブロッカーとリスクのあるマイルストーンを特定
- RAG（赤/黄/緑）ステータス付きの進捗レポート生成

**呼び出し元**: `/project-status`、`/weekly-report`

## スキル

スキルは、特定分野における Claude の専門知識を強化するドメイン知識パッケージです。

### コミュニケーション系

| スキル | 説明 | 主なパターン |
|-------|------|------------|
| **email-writing** | ビジネスメール作成 | トーンレベル、構成テンプレート、敬語、返信チェーン |
| **slack-etiquette** | 効果的なSlack活用 | スレッド運用、メンション戦略、リアクション、チャンネル規範 |
| **presentation-design** | プレゼン設計 | ストーリーアーク、スライドレイアウト、データ可視化 |
| **negotiation-prep** | 交渉準備 | BATNA分析、論点マッピング、譲歩計画 |

### 会議・意思決定系

| スキル | 説明 | 主なパターン |
|-------|------|------------|
| **meeting-design** | 会議設計 | アジェンダテンプレート、タイムボックス、決定プロトコル |
| **decision-frameworks** | 構造化された意思決定 | MECE分解、2×2マトリクス、意思決定ツリー |
| **facilitation** | 議論ファシリテーション | 収束技法、合意形成、対立解消 |

### 分析・戦略系

| スキル | 説明 | 主なパターン |
|-------|------|------------|
| **market-research** | 市場インテリジェンス | TAM/SAM/SOM、ポーターの5力、トレンド分析 |
| **competitive-analysis** | 競合環境分析 | SWOT、ベンチマーク、差別化マッピング |
| **financial-modeling** | 財務分析 | P/L分析、ROI計算、予算策定、ユニットエコノミクス |
| **data-storytelling** | データコミュニケーション | 可視化選択、ナラティブ構成、示唆抽出 |

### ドキュメント系

| スキル | 説明 | 主なパターン |
|-------|------|------------|
| **proposal-writing** | 提案書作成 | 課題→解決→効果、根拠構造化、CTA |
| **report-writing** | 報告書作成 | エグゼクティブサマリー、週次/月次フォーマット |
| **minutes-writing** | 議事録作成 | 決定事項、アクションアイテム、出席者、タイムライン |
| **sop-writing** | SOP作成 | プロセスフロー、チェックリスト、役割分担、例外処理 |

### プロジェクト管理系

|-------|------|------------|
| **project-planning** | プロジェクト構造化 | WBS、マイルストーン、依存関係、リソース配分 |
| **risk-management** | リスクプロセス | リスク台帳、確率×影響マトリクス、対策計画 |
| **okr-management** | OKR/KPIトラッキング | 目標設定、KR定義、進捗スコアリング |

## スキル（スラッシュコマンド）

### デイリーオペレーション

#### `/briefing`

カレンダー・タスク・メール・Slackを集約した朝のブリーフィング。

**フロー**:
1. 本日のカレンダーイベント取得 → 準備が必要な会議をハイライト
2. タスクDB照会 → 期限超過 + 本日期限 + 高優先度を表示
3. メールスキャン → 要対応アイテムを抽出（送信者の重要度加重）
4. Slackスキャン → メンション + DM + 重要チャンネルのサマリー
5. プロジェクト状況確認 → ブロッカーを表面化
6. 「今日やるべき3つのこと」を優先度順に提案

**引数**: なし

**出力例**:
```
## 朝のブリーフィング — 2026-03-20（木）

### カレンダー（6件）
- 09:00 Engineering Standup (15分) — 定例
- 10:00 Q1計画レビュー (60分) ⚠️ 準備必要
  → 参加者: CEO、CTO、PMリード
  → 前回は四半期実績を報告済み
- 13:00 田中さんとの1on1 (30分)
- ...

### タスク（緊急3件）
- ★★★ 予算提案書の提出 — 本日期限
- ★★★ PR #234 のレビュー — 1日超過
- ★★ 競合分析の更新 — 明日期限

### メール（要対応2件）
- CFO: 「本日中に予算数値が必要」 — 2時間前
- クライアント（山田様）: 「契約更新について質問」 — 5時間前

### Slack（メンション4件）
- #engineering: @あなた「デプロイ計画をレビューいただけますか？」
- ...

### 推奨フォーカス
1. 予算提案書を提出（本日期限）
2. Q1計画レビューの準備（10:00）
3. CFOメールに返信
```

#### `/review`

夕方の振り返りと翌日計画。

**フロー**:
1. 本日完了したタスクのサマリー
2. 参加した会議の主要アウトカム
3. 持ち越しタスクの特定
4. 明日の優先リスト生成
5. 保留中のフォローアップの下書き

---

#### `/inbox-triage`

メール + Slack メッセージの一括分類。

**引数**: `--email-only`、`--slack-only`、`--since "2h ago"`

---

### 会議コマンド

#### `/meeting-prep`

包括的な会議準備パッケージの生成。

**引数**: 会議名またはカレンダーイベント参照

**フロー**:
1. カレンダーで会議を検索 → 参加者、アジェンダ、場所
2. 同じ参加者との過去の会議を検索 → 主要な決定事項
3. 参加者との最近のメール/Slackコンテキストを検索
4. 生成: 論点、質問案、リスク指摘事項、必要資料

---

#### `/meeting-minutes`

生の入力から構造化された議事録を生成。

**引数**: 会議内容を貼り付けまたは記述

**出力フォーマット**:
```markdown
## 議事録: [タイトル]
**日付**: YYYY-MM-DD | **時間**: X分 | **出席者**: ...

### 決定事項
1. ...

### アクションアイテム
| # | アクション | 担当者 | 期限 |
|---|----------|-------|------|
| 1 | ...      | ...   | ...  |

### 主要な議論ポイント
- ...

### 次回会議
- 日程: ...
- アジェンダ: ...
```

---

#### `/meeting-followup`

議事録からアクションアイテムを抽出し、フォローアップを準備。

**フロー**:
1. 議事録からアクションアイテムを解析
2. 担当者ごとにグループ化
3. 各担当者へのSlackメッセージを下書き
4. Notion/Linearにタスクを作成
5. 期限のカレンダーリマインダーを設定

---

### ドキュメントコマンド

#### `/draft-email`

ビジネスメールの作成。

**引数**: 目的の説明、`--to`、`--tone [formal|professional|casual]`、`--lang [en|ja]`

**例**:
```
/draft-email "ベンダー会議を丁重にお断りし、来期を提案" --to vendor@example.com --tone formal --lang ja
```

---

#### `/draft-slack`

チャンネル/スレッドコンテキストを考慮したSlackメッセージの作成。

**引数**: 目的の説明、`--channel`、`--thread`

---

#### `/draft-proposal`

ビジネス提案書・企画書の生成。

**引数**: トピックの説明、`--audience [internal|client|investor]`

**出力構成**:
1. エグゼクティブサマリー
2. 課題の定義
3. 提案する解決策
4. 期待される効果
5. タイムラインとリソース
6. リスクと対策
7. 次のステップ

---

#### `/draft-report`

構造化されたビジネス報告書の生成。

**引数**: トピックの説明、`--type [weekly|monthly|project|executive]`

---

#### `/write-sop`

業務手順書（SOP）の作成。

**引数**: プロセスの説明

**出力構成**:
1. 目的と範囲
2. 役割と責任
3. 前提条件
4. 手順（ステップバイステップ）
5. 例外処理
6. 品質チェックリスト
7. 改訂履歴

---

### 分析・戦略コマンド

#### `/market-scan`

市場トレンドと競合環境のスキャン。

**引数**: 業界またはトピック、`--depth [quick|standard|deep]`

---

#### `/competitive-brief`

競合インテリジェンスブリーフの生成。

**引数**: 競合企業名または市場セグメント

---

#### `/data-insight`

データを分析し、実行可能な示唆を抽出。

**引数**: データの説明またはファイル参照、`--framework [swot|porter|bcg]`

---

#### `/decision-matrix`

加重意思決定マトリクスの作成。

**引数**: 意思決定のトピック

**フロー**:
1. 選択肢をヒアリング（またはコンテキストから提案）
2. 評価基準と重みをヒアリング
3. 各選択肢を各基準に対してスコアリング
4. 加重結果と推奨を提示
5. 各選択肢のリスク評価を付加
